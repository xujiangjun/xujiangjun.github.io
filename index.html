研发规范

  1. 对外提供的Dubbo服务设计时，需尽量保证接口的幂等性（新增&修改，查询本就是幂等的）。主要原因有：接口可能会重复调用、补偿机制、Dubbo调用存在重试机制。

  2. MQ消费者接收消息进行处理时需要保证消费的幂等性，避免因重复消费引起数据和流程异常。

  3. 批量处理时，如果没有强依赖性，且循环体内可能会抛出异常，则需要捕获循环体内的异常（不要重新抛出），否则后续任务会因为异常而不会被执行。

  4. 开发时尽量保证代码的可重用性（单一职责原则）。

  5. 创建表默认需要包含id、gmt_create和gmt_modified三个字段，修改时间默认设置为创建时间。

    解决方案：如果已经编译过了，则将服务器ip维护起来。若有修改代码，则服务器ip清空。

数据库基础

一个database可以有很多schema。此外一般线上会对应两个数据库，主库（master，主要进行写操作），从库（slave，主要进行读操作）。

数据库基础数据类型

数字（整数|小数）

  1. 整数通常只使用tinyint和int，其后面括号带的数字无任何意义，不影响所占用的字节数和存储范围。
  2. 枚举字段：使用tinyint，且可加上unsigned。
  3. decimal(n,m)简单介绍就是n>=m，n表示总位数（包含m，不包括小数点），m表示小数点后的位数（即精度）。n取值范围是（1~65）、m取值范围是（0~30），金额字段通常使用decimal(10,2)，根据需求定义n和m的值。

字符（定长|变长）

  1. 定长字符char(n)：n取值范围是（0~255），n的单位为char，当n为0时，只能存放null和空字符串（但是没意义）。
  2. 变长字符varchar(n)：最大存储空间为65535byte（64k），n的单位为char，故n的取值范围为（0~65535/m），m的取值与对应的字符集有关，不过n不要轻易超过255。
  3. 长度单位是char，存储单位是byte。
  4. 同等长度的字符长度，varchar会额外多占用一个字节。如：”ab”字符串varchar存储的话需要占用3bytes。

日期时间（日期|日期+时间）

  1. 日期时间推进使用datetime，不推荐使用date，在淘汽内部使用的数据表迁移工具对date支持不好。
  2. 如果日期时间只关心到月份，则推荐使用字符进行存储，如：char(6)存储201701。
  3. 记录修改时间什么时候赋值好？插入时即可进行赋值，以免为空。

Java面试题集

学习指南：https://github.com/Snailclimb/JavaGuide

进阶知识：https://github.com/doocs/advanced-java

面试经验：https://github.com/aylei/interview

  1. 监听器： Servlet API内置了大量监听器。
  2. 过滤器：Spring环境下，推荐使用OncePerRequestFilter。
  3. 拦截器：SpringMVC拦截器，深入分析拦截器，拦截器可以有三种实现方式：实现HandlerInterceptor接口、实现MethodInterceptor接口（参考ServiceWrapper）或继承HandlerInterceptorAdapter抽象类（参考saint的TokenInterceptor），preHandle方法中第三个参数handler可以通过instance of转化为HandlerMethod，由此可见拦截器是针对方法的。拦截器是单例的，是线程不安全的。多线程环境下，共享变量可考虑使用线程本地变量ThreadLocal。拦截器中经常使用HttpServletRequest的getContextPath()、getServletPath()、getRequestURI()方法。
  4. 引用关系：强引用、软引用、弱引用和虚引用。
  5. 多线程（老马说编程）：线程、线程池、Synchronized、Volatile、ReentrantLock、CAS（CompareAndSet）、原子变量、ThreadLocal、NamedThreadLocal（Spring提供的一个命名的ThreadLocal实现，相比ThreadLocal多了个name属性，toString()方法打印出name的值），sleep()、yield()、join()，线程协作的wait()、notify()和notifyAll()方法需要在Synchronized同步块内使用。Volatile如何保证内存可见性。
  6. 注解、泛型、反射
  7. 设计模式：工厂方法模式、抽象工厂模式、观察者模式、适配器模式、策略模式、装饰器模式、代理模式
  8. 动态代理：基于接口的JDK动态代理、基于类的CGLib动态代理
  9. Spring Bean的生命周期
  10. Spring IOC
  11. Spring AOP
  12. 集合类：ArrayList、ArrayList的默认初始大小是10，HashMap默认初始大小为（必须是2倍数）
  13. Redis
  14. JVM参数配置 -Xms20m -Xmx20m -Xmn20m -XX:PermSize=128m -XX:MaxPermSize=128 -Xss128k
  15. 正则表达式：正则表达式语法、利用正则检查是否时间格式是否正确、阅读String源码中replace与replaceAll方法
  16. String源码解析：replace(String, String)与replaceAll()是通过正则表达式实现的，replaceAll第一个参数可以传入正则表达式。substring()方法就是根据参数，调用构造方法String(char value[], int offset, int count)新建了一个字符串。
  17. HashMap源码解析：HashMap创建Entry数组是在put时。通过hash值得到数组下标是通过hash值与table的长度决定、HashMap第一次put时，如果设置的容量不是2次幂，则会将容量向上取最近的2次幂。此后的put操作，如果hashMap的size大于等于阈值，则会将容量扩大至原来两倍。
  18. Java IO与NIO的区别
  19. CountDownLatch这个类能够使一个线程等待其他线程完成各自的工作后再执行。
  20. fail-fast机制是Java集合中的一种错误机制当多个线程对同一个集合的内容进行操作时，就可能会产生fail-fast事件。例如：当某一个线程A通过iterator去遍历某集合的过程中，若该集合的内容被其他线程所改变了；那么线程A访问集合时，就会抛出ConcurrentModificationException异常，产生fail-fast事件。

  22. CountDownLatch
  23. Unix下的五种IO模型
  24. ThreadLocalRandom

Spring简介

Spring 对 Java 版本的要求

Spring 4.0 基于 Java 6.0，全面支持 Java 8.0。运行 Spring 4.0 必须使用 Java 6.0 以上版本。

Spring 4.0 新特性

  1. DateTimeFormat注解可以通过配置格式化”yyyy-MM-dd”格式的日期字符串。
  2. 空指针终结者Optional<>、依赖注入和SpringMVC参数接收可选化。
  3. 核心容器增强：支持泛型依赖注入、对Cglib类代理不再要求必须有空参构造器。
  4. 从 Spring 4.0 开始，SpringMVC 基于 Servlet 3.0 开发。
  5. 使用 WebSocket 需要考虑浏览器版本（IE<10 不支持），目前主流的浏览器都能很好地支持WebSocket。

Spring 子项目

Spring Boot、Spring Cloud、Spring Data、Spring Security、Spring Session

如何获取 Spring

目前，Spring 不再提供直接下载方式，只能使用Maven或Gradle构建来下载Spring的构建包。
